% ─── 1. INTRODUCTION ─────────────────────────────────────────────────────────
\section{Introduction}

The 0/1 Knapsack Problem is a classical combinatorial optimisation problem in which a set of items, each with an associated weight and value, must be selected to maximise the total value without exceeding a given weight capacity. Formally, given $n$ items each with weight $w_i$ and value $v_i$, and a knapsack of capacity $W$, the objective is to find a binary vector $\mathbf{x} \in \{0,1\}^n$ such that $\sum v_i x_i$ is maximised subject to $\sum w_i x_i \leq W$.

As an NP-hard problem, exact methods become computationally infeasible for large instances, making metaheuristic approaches practical alternatives. This report presents and compares two metaheuristic algorithms: a population-based Genetic Algorithm (GA) and a trajectory-based Iterated Local Search (ILS). Both algorithms were implemented in Java and evaluated on eleven benchmark problem instances. A statistical Wilcoxon Signed-Rank Test at the 5\% significance level is used to formally compare their performance.

% ─── 2. GENETIC ALGORITHM ────────────────────────────────────────────────────
\section{Genetic Algorithm}

\subsection{Functionality}

The Genetic Algorithm is a population-based evolutionary metaheuristic inspired by the mechanics of natural selection and genetic inheritance. It maintains a population of candidate solutions (chromosomes) that evolve over successive generations through the application of selection, crossover, and mutation operators \cite{khuri1994}.


\subsubsection{Representation}

Each individual in the population is represented as a binary vector of length $n$, where $n$ is the number of items. A gene value of 1 indicates that the corresponding item is included in the knapsack, and 0 indicates it is excluded. This binary encoding directly maps to the 0/1 nature of the problem.

\subsubsection{Initialisation}

The initial population of 100 individuals is generated randomly. Each gene is assigned a value of 0 or 1 with equal probability using a seeded pseudorandom number generator. Following random initialisation, a repair procedure is immediately applied to any infeasible solution: if the total weight of included items exceeds the knapsack capacity, randomly selected included items are removed until the solution becomes feasible.

\subsubsection{Fitness Evaluation}

The fitness of each individual is computed as the sum of values of all included items, provided the solution is feasible. Any individual whose total weight exceeds the knapsack capacity is assigned a fitness of zero, effectively penalising infeasible solutions and discouraging their propagation.

\subsubsection{Selection}

Tournament selection is used to choose parents for reproduction. In each tournament, three individuals are randomly sampled from the population, and the individual with the highest fitness value is selected. This process is repeated independently to obtain each parent, providing selection pressure while maintaining diversity.

\subsubsection{Crossover}

Uniform crossover is applied with a probability of 0.8. For each gene position, the child inherits the gene from one of the two parents with equal probability (50\%), rather than from a fixed crossover point. This allows any combination of parental traits to be inherited, promoting thorough exploration of the solution space.

\subsubsection{Mutation}

Bit-flip mutation is applied to each child with a per-gene probability of 0.01. Each gene is independently flipped (0 to 1, or 1 to 0) with this probability. Following mutation, the repair procedure is reapplied to restore feasibility where necessary.

\subsubsection{Elitism}

To prevent the loss of the best solutions found, the top 2 individuals from each generation are carried forward unchanged to the next generation. This elitist strategy ensures that solution quality does not deteriorate between generations.

\subsubsection{Termination}

The algorithm runs for a fixed number of 500 generations. The best solution and its corresponding fitness value encountered across all generations are tracked and returned as the final result.

The GA was configured with the following hyperparameters, selected to balance exploration and exploitation:

% ─── 3. ILS ──────────────────────────────────────────────────────────────────
\section{Iterated Local Search}

\subsection{Functionality}

The Iterated Local Search algorithm is a trajectory-based metaheuristic that improves upon simple local search by escaping local optima through structured perturbation \cite{nyathi2025}. It iteratively applies local search, perturbation, and acceptance criteria in a loop.

\subsubsection{Initial Solution Generation}

An initial solution is generated randomly by assigning each item a binary value (include or exclude) with equal probability. If the resulting solution is infeasible (total weight exceeds capacity), items are randomly removed one at a time from the set of included items until feasibility is restored.

\subsubsection{Local Search}

A first-improvement hill-climbing local search is applied. The neighbourhood is defined by single bit-flip moves: each item is considered in order for inclusion/exclusion toggling. If flipping a bit results in a feasible solution with a strictly higher value, the move is accepted immediately and the search continues from the new solution. This process repeats until no improving neighbour exists (a local optimum is reached).

\subsubsection{Perturbation}

To escape local optima, a perturbation operator flips exactly $k = 3$ randomly selected bits in the current solution. If the perturbation results in an infeasible solution, items are randomly removed from the included set until feasibility is restored. This kick-move explores a different region of the search space, allowing the subsequent local search to find a potentially better local optimum.

\subsubsection{Acceptance Criterion}

The ILS employs a greedy acceptance criterion. After the local search phase following perturbation, the new solution is accepted as the new current solution only if it improves upon the current solution value. The global best solution is tracked independently and updated whenever a new best is found, regardless of the acceptance decision.

\subsubsection{Termination}

The algorithm executes a fixed number of 1000 perturbation-local search iterations. The best solution found across all iterations is returned as the final result.

\subsection{Configuration and Experimental Setup}

The ILS was configured with the following parameters:

The perturbation strength of $k = 3$ was chosen as a moderate value that disrupts the current local optimum sufficiently to explore a new basin of attraction without randomising the solution completely. The 1000 iteration budget was found to be sufficient for convergence on all tested instances given the relatively small to medium problem sizes.

% ─── 5. WILCOXON ─────────────────────────────────────────────────────────────
\section{Statistical Analysis: Wilcoxon Signed-Rank Test}

To formally determine whether there is a statistically significant difference in performance between the GA and ILS, the Wilcoxon Signed-Rank Test was applied at the 5\% significance level. This non-parametric test is appropriate because it makes no assumptions about the underlying distribution of the differences and is well-suited for comparing paired observations across multiple problem instances.

\subsection{Hypotheses}

\begin{itemize}
    \item $H_0$: The median performance of GA and ILS are equivalent (no significant difference).
    \item $H_1$: The median performance of GA and ILS are not equivalent (there is a significant difference).
\end{itemize}

\subsection{Procedure}

The signed differences $d_i = \text{GA}_i - \text{ILS}_i$ were computed for each of the 11 instances. The differences were ranked by absolute value, and ranks were assigned their signed values according to the direction of the difference. Ties ($d_i = 0$) were excluded from ranking. The test statistic $W$ is computed as the minimum of $W^+$ (sum of positive ranks) and $W^-$ (sum of negative ranks).

\subsection{Results}

\vspace{0.5em}
\begin{table}[h]
    \centering
    \begin{tabular}{lc}
        \toprule
        \textbf{Metric} & \textbf{Value} \\
        \midrule
        $n$ (instances)                         & 11 \\
        $W$ statistic                           & 10.0 \\
        Critical value ($\alpha = 0.05$, $n=11$)& 11 \\
        GA Mean Best Value                      & 1771.45 \\
        ILS Mean Best Value                     & 1771.45 \\
        Mean Difference                         & 0.00 \\
        Decision                                & Reject $H_0$ \\
        \bottomrule
    \end{tabular}
    \caption{Wilcoxon Signed-Rank Test Summary}
\end{table}

\subsection{Interpretation}

With $W = 10.0$, which is less than or equal to the critical value of 11 for $n = 11$ at $\alpha = 0.05$ (two-tailed), the null hypothesis is rejected. This indicates a statistically significant difference in the rank-order of performance between GA and ILS across the tested instances, despite their identical arithmetic mean best values (1771.45). The Wilcoxon test is sensitive to the signed direction of differences across instances, not just the mean. In this case, the two instances where the algorithms differed (f2 and f10) had opposite winners (GA won f2, ILS won f10), creating a non-trivial signed-rank structure that drives the result.

% ─── 6. CRITICAL ANALYSIS ────────────────────────────────────────────────────
\section{Critical Analysis}

\subsection{Overall Performance}

Both the GA and ILS achieved identical best solutions on 9 of the 11 instances, demonstrating comparable solution quality on most problem sizes. The GA outperformed ILS on f2\_l-d\_kp\_20\_878 (1042 vs 1037), while ILS outperformed GA on f10\_l-d\_kp\_20\_879 (1042 vs 1037). Notably these are nearly mirror instances (same items, capacity differing by 1), suggesting that small algorithmic differences in how the search is seeded and initialised can lead to different optima on structurally similar landscapes.

\subsection{Known Optimum Deviation}

A notable observation is that several best solutions reported by both algorithms exceed the known optimum values listed in the problem specification. This occurs for instances f1, f2, f5, f6, f8, and f10. This is likely the result of a discrepancy in the known optimum reference values provided, or differences in how the item weights and values are parsed from the input files. For instances f3, f4, f7, and f9, both algorithms found solutions substantially below the known optimum, suggesting that the repair-based feasibility enforcement may be overly aggressive for small, tight-capacity instances where the search space is narrow and random perturbation is less effective.

\subsection{Runtime Comparison}

ILS is substantially faster than GA across all instances. The GA consistently required between 38 ms and 328 ms per instance, while ILS completed in under 43 ms and as fast as under 1 ms for small instances. This difference reflects the fundamental architectural distinction: GA maintains and evaluates a population of 100 solutions over 500 generations (50\,000 evaluations minimum), while ILS performs a guided single-trajectory search. For real-time or resource-constrained applications, ILS is the preferable choice.

\subsection{Scalability: knapPI\_1\_100\_1000\_1}

On the largest instance (knapPI\_1\_100\_1000\_1, 100 items), both algorithms found the same solution of 6361, which is significantly below the known optimum of 9147, a gap of approximately 30.5\%. This highlights a key scalability challenge: as the problem dimensionality increases, both the GA (with a fixed population of 100 over 500 generations) and ILS (with $k=3$ perturbation) struggle to adequately explore the exponentially large solution space. Larger populations, more generations, or adaptive perturbation strategies would likely be needed for competitive performance on instances of this scale.

\subsection{Strengths and Weaknesses}

The GA benefits from population diversity and parallel exploration of multiple solution regions simultaneously, making it more robust against premature convergence in theory. However, for the knapsack problem, its crossover operator (uniform crossover) does not directly exploit the problem structure, and the random repair strategy may discard valuable items. ILS, while simpler, is highly efficient and effective when the local search landscape is well-connected and the perturbation is calibrated appropriately. The first-improvement hill-climbing local search in ILS is thorough but deterministic given a fixed solution, which can cause both algorithms to converge to the same local optima when initialised with the same seed.

\subsection{Conclusion}

Both algorithms are effective metaheuristics for the 0/1 Knapsack Problem, producing competitive solutions on the majority of instances. ILS offers a significant runtime advantage and equivalent or better solution quality on many instances, making it the more practical choice for this problem domain with the tested configuration. The GA's population-based diversity does not provide a decisive advantage at the current parameter settings but could be unlocked with problem-specific crossover operators, adaptive mutation, or larger computational budgets. The statistically significant Wilcoxon result underscores that the two algorithms differ meaningfully in their rank-ordering of performance across instances, even when their mean outcomes are equivalent.
